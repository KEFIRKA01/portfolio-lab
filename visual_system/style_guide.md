# Visual System

## Цель

Профили и кейсы должны выглядеть не "как очередной шаблонный фрилансер", а как зрелый технический специалист с системным подходом.

## Визуальный принцип

- светлый фон;
- глубокие тёмные блоки только для акцентов;
- насыщенные, но не кислотные градиенты;
- много воздуха;
- жёсткая сетка;
- техничные карточки, а не "маркетинговый шум";
- один визуальный мотив: product engineering, а не абстрактный digital.

## Цвета

- `#0F172A` — основной тёмный текст
- `#F8FAFC` — фон
- `#E2E8F0` — границы и разделители
- `#0EA5E9` — технологический акцент
- `#14B8A6` — интеграции / стабильность
- `#F97316` — e-commerce / business tone
- `#8B5CF6` — кабинет / интерфейсы
- `#F43F5E` — rescue / bugfix акцент

## Формат обложек

- размер: `1600x900`;
- слева название проекта и одна строка ценности;
- справа 3-4 чипа по стеку и роли проекта;
- снизу подпись формата: `Demo project / GitHub / Case`;
- без лишних мокапов и псевдо-3D.

## Как выглядеть сильнее конкурентов

- не делать случайные превью из стоков;
- не вставлять на обложку длинное описание;
- использовать один стиль для всей линейки;
- у каждого кейса должен быть свой цветовой акцент, но единая система;
- показывать не "креатив", а инженерную собранность.
