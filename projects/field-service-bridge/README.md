`Мост выездного сервиса` показывает, как Telegram может стать рабочей системой для координации полевых специалистов: распределять выезды, держать SLA, учитывать запчасти, фиксировать результат визита и быстро поднимать эскалацию, если процесс ломается.

## Что показывает проект

- прикладной field service workflow вместо абстрактного “бота для заявок”;
- работу с маршрутами визитов, статусами, повторными выездами и запасными частями;
- сервисное мышление: контроль SLA, исключений и post-visit отчётности;
- архитектуру, подходящую для сервисных компаний, инсталляций, ремонта, выездной поддержки и технического обслуживания.

## Для каких задач подходит

- распределение выездных инженеров и мастеров;
- управление срочными заявками и повторными визитами;
- резервирование и контроль запчастей;
- Telegram-интерфейс для координации сервисной команды;
- сокращение ручной диспетчеризации и потерь на эскалациях.

## Ключевые сценарии

- новая заявка на выезд;
- срочный ремонт с риском SLA;
- отсутствие нужной запчасти;
- повторный визит после диагностики;
- обслуживание VIP-клиента с повышенным приоритетом.

## Быстрый старт

```bash
pip install -r requirements.txt
python -m bot.main
uvicorn bot.api:create_app --factory --reload
```

## Почему это сильный кейс

- показывает зрелость в операционных и сервисных процессах, которые редко хорошо раскрыты в портфолио;
- помогает продавать заказы про диспетчеризацию, сервис, полевые команды и SLA;
- доказывает, что профиль умеет работать не только с интерфейсом, но и с тяжёлой прикладной логикой.
