# Кейс: FlowDesk CRM Hub

## Задача

Показать реальный прикладной контур для задач по лидам и интеграциям: принять заявку, нормализовать данные, отфильтровать дубли, передать данные дальше и сохранить историю действий.

## Решение

Собран демонстрационный сервис приёма лидов с API, доменной логикой, журналом событий и поддержкой сценариев с webhook и колбэками. Проект специально устроен так, чтобы его можно было показывать и как доказательную базу по backend, и как доказательную базу по CRM/API-интеграциям.

## Стек

- Python
- FastAPI
- Pydantic
- JSON seed-данные

## Что сделано

- модель лида с сигнатурой для дедупликации;
- классификация канала и оценка источника;
- журнал событий по заявке;
- webhook-эндпоинт для внешнего источника;
- эндпоинт платёжного колбэка;
- операторский список лидов и базовый статусный контур;
- seed-набор реалистичных демо-данных.

## Практическая польза

Проект показывает, что я умею работать не только с "получили POST и сохранили в базу", а с реальным процессом: защита от дублей, маршрутизация, операционный контур и наблюдаемость событий.

## Что проект доказывает

- CRM/API интеграции;
- webhook- и колбэк-потоки;
- бизнес-логика вокруг форм и заявок;
- backend-мышление и системность.
