# Кейс: Командный центр продажи билетов

## Задача

Нужно продавать билеты прямо в Telegram, удерживать места, быстро добирать очередь ожидания и контролировать доступ на площадке.

## Решение

В проекте `Telegram` выступает как рабочая поверхность для клиента и команды, а не только как чат.

- удержание мест
- очередь ожидания
- пакеты и апсейлы
- статусы билетов
- сценарий проверки доступа

## Что сделано

- спроектирована жизненная модель объекта `билетный заказ`;
- собраны маршруты для роли `координатор события`;
- добавлены сценарии: Покупка билета, Повышение до VIP, Очередь ожидания, Групповой заказ, Проверка доступа;
- добавлены риск-сигналы: последние места, групповой заказ, возврат, повышение класса, массовый вход.

## Результат

Показывает продуктовый Telegram-контур: продажи, лимиты, очередь, доступ и работу команды события.
