# Кейс: Защитник входов в Telegram

## Задача

Нужно подтверждать входы через Telegram, отсекать подозрительные сессии и давать безопасный путь для доверенных устройств.

## Решение

В проекте `Telegram` выступает как рабочая поверхность для клиента и команды, а не только как чат.

- подтверждение входа
- риск-скоринг устройства
- усиленная проверка
- журнал сессий
- отзыв активной сессии

## Что сделано

- спроектирована жизненная модель объекта `сессия входа`;
- собраны маршруты для роли `аналитик безопасности`;
- добавлены сценарии: Новый девайс, Подозрительный IP, Повышенный риск, Доверенный вход, Ревизия активных сессий;
- добавлены риск-сигналы: новая геолокация, ночной вход, финансовый раздел, анонимайзер, много неудачных попыток.

## Результат

Показывает Telegram-решение с акцентом на безопасность: MFA-логика, риск-сигналы и расследования.
