# Кейс: EventMesh Booking Router

## Задача

Показать, что интеграции и API — это не только CRM и лиды, но и более сложные событийные сценарии с бронированиями, слотами и несколькими провайдерами.

## Решение

Собран демонстрационный маршрутизатор бронирований, который принимает запрос, проверяет провайдеров, слоты, ограничения по каналу и приоритету, а затем предлагает лучший маршрут или резервный маршрут.

## Что сделано

- модель запроса на бронирование;
- маршрутизация по возможностям;
- контроль конфликтов слотов;
- резервный маршрут chain;
- логика SLA и срочности;
- API для preview маршрута и фиксации брони.

## Что проект доказывает

- зрелую интеграционную логику;
- event-driven мышление;
- работу с необычными задачами планирования.
