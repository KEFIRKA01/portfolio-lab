# Кейс: Консьерж просмотров недвижимости

## Задача

Нужно быстро квалифицировать интерес клиента к недвижимости, собрать базовые параметры сделки и не терять лиды на этапе записи на просмотр.

## Решение

В проекте `Telegram` выступает как рабочая поверхность для клиента и команды, а не только как чат.

- квалификация бюджета
- подборка объектов
- слоты просмотров
- маршрут агента
- ипотечный сценарий

## Что сделано

- спроектирована жизненная модель объекта `запрос на объект`;
- собраны маршруты для роли `агент по недвижимости`;
- добавлены сценарии: Подбор аренды, Покупка квартиры, Коммерческое помещение, Ипотечный пречек, Маршрут на несколько просмотров;
- добавлены риск-сигналы: плавающий бюджет, несколько участников сделки, горящий просмотр, документы не готовы, высокая конкуренция.

## Результат

Раскрывает бизнес-логику лидогенерации, квалификации, подборов и маршрутов просмотров.
